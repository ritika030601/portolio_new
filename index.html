My Process

The Hybrid Edge

Design that speaks developer, code that respects design.

Having worked as an engineer, I know what it actually takes to build something. That means I design with the real constraints in mind - so what looks good on screen also holds up once it's in production.

01
Research & Reality

Rooted in Reality

I try to understand both what users need and what the system can realistically support, so the solutions aren't designed in isolation. Interviews, data, and behavior patterns help me get to the real reason behind a problem, not just what's visible on the surface.

02
Strategy & Logic

Feasible Innovation

I think about edge cases, data, and layout constraints from the start, because I've worked as an engineer and know what's realistic to build. That means the strategies I hand off are ones a team can actually run with, without redoing the plan halfway through.

03
Precision & Performance

Production-Ready Design

Every prototype is built with design systems and consistent components, so it translates cleanly into code instead of staying a static mockup. It's high-fidelity, testable, and ready to build on - because the work doesn't stop the moment something ships.
